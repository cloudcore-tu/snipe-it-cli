# ドキュメント整合性チェック

コード変更に対して、関連ドキュメントの更新漏れがないかを検証する。

## チェック対象

| ドキュメント | 更新条件 |
|-------------|----------|
| `.claude/CLAUDE.md` | ツール・ライブラリ変更、ビルド設定変更、ディレクトリ構成変更、設計方針変更、設定項目変更 |
| `docs/adr/` | 設計判断の変更・新規追加 |
| `CHANGELOG.md` | 機能追加・変更・削除 |

## ワークフロー

### Step 1: 変更差分の取得

```bash
git diff HEAD
git diff HEAD~1  # コミット後のチェック用
```

### Step 2: 変更カテゴリの判定

差分から以下を特定する:

- **依存関係の変更**: `go.mod`, `mise.toml` の変更
- **設定・ビルドの変更**: ビルドコマンド、lint 設定、テスト設定の変更
- **ディレクトリ構成の変更**: 新ディレクトリの追加、ファイル移動
- **設計方針の変更**: アーキテクチャに影響するコード変更
- **機能の変更**: ユーザーに見える振る舞いの追加・変更・削除

### Step 3: 各ドキュメントの整合性確認

変更カテゴリに応じて、対応するドキュメントを読み込み、内容が最新かどうかを検証する。

確認ポイント:
- CLAUDE.md のツール一覧に新しいライブラリが反映されているか
- CLAUDE.md のディレクトリ構成に新しいパスが反映されているか
- CHANGELOG.md に機能変更が記載されているか
- ADR の決定事項と実装が一致しているか

### Step 4: 結果報告

**整合している場合**: 確認した項目を簡潔に一覧で報告

**更新漏れがある場合**: どのドキュメントに何が不足しているかを具体的に指摘し、更新すべき内容の案を提示
