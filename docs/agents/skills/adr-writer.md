# adr-writer

タスクが設計判断を導入または記録する場合に、`docs/adr/` 配下へ新しい ADR を追加する。

## 手順

1. `docs/adr/` の既存 ADR を見て次の番号を決める。
2. `NNN-kebab-case-title.md` を作る。
3. 次の template を使う。

```markdown
# ADR-NNN: Title

- **ステータス**: 提案 | 承認 | 却下 | 廃止
- **日付**: YYYY-MM-DD

## コンテキスト

## 検討した選択肢

## 決定

## 結果

## 未決事項
```

## ルール

- デフォルト status は `提案`。
- コンテキストは推測でなく事実を書く。
- 何を選んだかだけでなく、なぜその案が勝ったかを書く。
